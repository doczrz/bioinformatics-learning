# 0.1｜为什么生物信息学常在 Linux 上运行？R、Python 与 Codex 分别做什么？

假设你拿到一批单细胞测序数据，希望回答：“疾病组中哪些细胞发生了变化？”

在得到答案之前，可能需要完成许多工作：

> 下载数据 → 检查文件 → 运行质量控制 → 比对或计数 → 整理表达矩阵 → 统计分析 → 绘图 → 解释结果

这些步骤并不全部由同一种软件完成。有些程序在 Linux 中运行，有些分析使用 R 或 Python 编写，Codex 等 AI 工具则可以帮助我们生成代码、执行命令和检查结果。

初学者最容易产生的误解，是把 Linux、R、Python 和 Codex 看成四个互相竞争的分析软件。实际上，它们处在不同层级：

| 名称 | 它是什么 | 在生物信息学中的常见作用 |
| --- | --- | --- |
| Linux | 操作系统 | 管理文件、运行生物信息学软件、连接服务器和组织分析流程 |
| Bash | Linux 中常见的命令行环境 | 输入命令、连接多个程序、批量处理文件 |
| R | 编程语言和统计计算环境 | 统计分析、数据可视化、转录组和单细胞分析 |
| Python | 通用编程语言 | 数据处理、流程自动化、机器学习和组学分析 |
| Codex | 可以操作代码、文件和工具的 AI 智能体 | 协助编写、解释、运行和检查分析代码 |

可以暂时把它们的关系理解为：Linux 提供计算环境；Bash、R 和 Python 在这个环境中承担不同任务；Codex 帮助研究者组织、执行和检查工作，但不替代研究者对问题与证据的判断。

这张图只表示常见分工，不是硬性规定。R 和 Python 也能在 Windows、macOS 上运行；Linux 也不是只能使用命令行。

## 本节要解决的三个问题

读完本节后，你应该能够回答：

1. 为什么许多生物信息学流程选择 Linux？
2. R 和 Python 各自适合做什么，应该如何选择？
3. 当 Codex 已经可以直接写代码和运行分析时，为什么仍要学习这些基础知识？

本节不要求你安装 Linux，也不要求你记住任何命令。

## 为什么生物信息学经常使用 Linux？

### 原因一：许多生物信息学软件首先面向 Linux

大量测序数据处理工具是在 Linux 或类似的 Unix 环境中开发和测试的。有些软件也提供 macOS 版本，但不直接支持 Windows。

例如，Bioconda 提供了大量生物医学研究软件包，其官方文档目前支持 Linux 和 macOS，而不直接支持 Windows。10x Genomics 的 Cell Ranger 官方系统要求也以 Linux 为运行环境。

[Bioconda 官方文档](https://bioconda.github.io/index.html) · [Cell Ranger 官方系统要求](https://www.10xgenomics.com/support/software/cell-ranger/downloads/cr-system-requirements)

这不表示 Windows 不能用于生物信息学。真正的意思是：如果一个工具只在 Linux 上经过正式支持，那么直接使用 Linux 环境通常更容易获得与官方文档一致的行为。

### 原因二：服务器和大型计算资源常见 Linux 环境

普通笔记本适合阅读教程、检查小型数据和学习分析方法，但测序原始数据可能需要较多内存、存储空间和处理器核心。

实验室服务器、高性能计算集群（high-performance computing cluster，HPC）和许多云计算实例通常提供 Linux 环境。学习 Linux 后，同一套命令和目录结构往往可以从个人电脑迁移到服务器。

不过，Linux 本身不会让计算机凭空变快。软件能否运行，仍取决于处理器、内存、硬盘空间和数据规模。即使在 Windows 中安装了 Linux 环境，如果电脑的内存不足，也不能可靠地处理需要大量内存的原始测序数据。

### 原因三：命令容易组合、记录和重复

图形界面适合探索，但连续点击许多按钮后，我们可能很难准确回答：

- 当时选择了哪个输入文件？
- 使用了哪些参数？
- 软件版本是什么？
- 哪一步出现了警告？
- 如何对另外二十个样本重复同样操作？

在 Linux 命令行中，一次分析可以保存为脚本。脚本是一份由计算机执行的操作清单。它可以被检查、修改和重新运行，也可以交给其他研究者复现。

这仍不代表“保存了脚本就一定可重复”。参考基因组、注释文件、软件版本、样本信息和随机种子等因素也可能改变结果。

### 原因四：Linux 有成熟的文件和权限管理方式

生物信息学分析经常包含大量文件：FASTQ、BAM、表达矩阵、样本表、日志和结果图。如果没有清楚的目录结构，一个程序可能读取错误样本，或者把新结果覆盖到旧结果上。

Linux 提供了明确的文件路径、用户权限和运行日志机制。学习这些内容的目的不是背诵命令，而是知道：

- 数据现在在哪里；
- 程序正在读取什么；
- 结果将写到哪里；
- 哪些文件可以修改；
- 失败后应该查看哪份日志。

## Windows 用户是不是必须重装电脑？

不是。

Windows Subsystem for Linux（WSL，适用于 Linux 的 Windows 子系统）可以让 Windows 用户在不安装双系统的情况下运行 Linux 发行版、Bash 和 Linux 命令行程序。

[Microsoft WSL 官方说明](https://learn.microsoft.com/en-us/windows/wsl/about)

初学阶段可以把 WSL 理解为“Windows 电脑中的 Linux 工作环境”。以后如果需要使用实验室服务器，很多基本概念仍然适用。

但 WSL 与远程服务器不是同一个东西：

- WSL 使用的是当前电脑的计算资源；
- 服务器使用的是远程机器的资源；
- HPC 还可能通过任务调度系统安排多人提交的计算任务；
- 云平台可能按存储量和计算时间收费。

因此，本课程后续会分别说明“命令在哪里运行”和“数据存在哪里”，不会只给出一段无法判断运行位置的代码。

## R 是什么？

R 是一种编程语言，也是一套面向统计计算与绘图的环境。R 官方将其定位为用于统计计算和图形展示的语言与环境。

[R Project 官方介绍](https://www.r-project.org/about.html)

在生物信息学中，R 的重要优势之一是 Bioconductor。Bioconductor 是以 R 为基础的开源生物信息学项目，覆盖基因组、转录组、流式细胞术等数据，并强调统计方法、文档和可重复研究。

[Bioconductor 官方介绍](https://bioconductor.org/about/index.html)

在单细胞分析中，Seurat 是常见的 R 软件包，可以进行质量控制、数据探索、聚类、整合和可视化。

[Seurat 官方文档](https://satijalab.org/seurat/)

R 的常见优势包括：

- 统计分析生态成熟；
- Bioconductor 提供大量组学分析软件包；
- 数据分析、统计模型和绘图可以写在同一份脚本或报告中；
- 很多生物医学论文的方法和教程直接提供 R 代码。

R 也有需要适应的地方：

- 软件包可能要求特定的 R 或 Bioconductor 版本；
- 不同软件包可能使用不同的数据对象；
- 大型数据分析需要理解内存占用，不能只反复复制对象；
- 能画出图不代表统计设计正确。

## Python 是什么？

Python 是一种通用编程语言。除了科学计算，它也广泛用于软件开发、自动化、网络服务、机器学习和教育。Python 官方的软件包索引提供大量第三方模块。

[Python 官方介绍](https://www.python.org/about/)

在单细胞领域，Scanpy 是基于 Python 的分析工具，可以完成预处理、可视化、聚类、轨迹推断和差异表达等工作，并与 AnnData 数据对象配合使用。

[Scanpy 官方文档](https://scanpy.readthedocs.io/)

Python 的常见优势包括：

- 适合把数据下载、文件处理、分析和自动化连接成完整流程；
- 科学计算、机器学习和深度学习生态广泛；
- 容易与数据库、网页服务和其他程序连接；
- 适合开发可以重复运行的软件和分析工具。

Python 也有需要适应的地方：

- 软件包数量很多，相同问题可能有多套做法；
- Python、包和底层库的版本需要协调；
- 交互式笔记本如果执行顺序混乱，可能出现“这次能运行、重新打开却不能复现”的情况；
- 通用语言提供了很大自由度，也要求使用者主动管理代码结构和数据状态。

## R 和 Python，到底哪个更好？

没有脱离任务的统一答案。

| 比较角度 | R | Python |
| --- | --- | --- |
| 最初的核心定位 | 统计计算与图形 | 通用编程 |
| 典型生物信息学生态 | Bioconductor、Seurat | NumPy、pandas、scikit-learn、Scanpy/scverse |
| 常见优势场景 | 统计建模、差异分析、生物统计、论文图表 | 自动化、机器学习、数据工程、开发分析工具 |
| 常见学习困难 | 包版本、数据对象和统计概念 | 环境管理、工程结构和大量可选库 |
| 是否能做单细胞分析 | 可以 | 可以 |
| 是否能运行在 Linux | 可以 | 可以 |

下面几种说法都过于简单：

- “R 只能画图。”
- “Python 一定比 R 快。”
- “Python 能做所有事情，所以不需要 R。”
- “生物信息学论文常用 R，所以不需要 Python。”

真正的运行速度取决于算法、数据结构、底层实现、内存和是否使用并行计算，而不只是语言名称。许多 R 和 Python 软件包的核心计算也会调用 C、C++、Fortran 或其他高性能代码。

更实用的选择方式是先问：

1. 研究问题推荐的成熟方法在哪个生态中？
2. 官方教程和原始论文提供什么实现？
3. 团队使用什么数据对象和流程？
4. 结果是否需要与现有项目衔接？
5. 自己能否解释和验证每一步？

一个真实项目也可能同时使用两种语言：Linux 工具生成表达矩阵，R 完成统计分析，Python 进行额外的机器学习或自动化。

## Codex 已经能直接分析了，为什么还需要这套教程？

Codex 可以读取和修改文件、编写代码、运行任务，并协助构建自动化工作流。OpenAI 对 Codex 的当前介绍也明确强调，它能跨越文件和工具执行实际工作。

[OpenAI Codex 介绍](https://openai.com/academy/what-is-codex/)

这确实改变了学习方式。初学者不再需要先记住大量语法，才能开始检查数据或尝试一个分析流程。

但这不等于可以跳过生物信息学知识。

### 能成功运行，不等于分析正确

一段代码可能没有报错，也生成了漂亮的图，但仍可能存在这些问题：

- 疾病组和对照组没有独立生物学重复；
- 把每个细胞错误地当成独立样本；
- 使用了不匹配的参考基因组；
- 过滤阈值删除了真正重要的低 RNA 细胞；
- 批次和实验条件完全重合；
- 图中出现的两个细胞群只是计算上相近，并不证明它们存在因果关系。

这些问题往往不是语法错误，因此不能只靠“代码是否运行成功”发现。

### AI 需要一个能够判断结果的人

可以把 Codex 暂时理解为一位速度很快、能够操作工具的研究助手。你可以让它整理文件、解释报错、编写脚本和执行检查，但你仍然需要告诉它：

- 真正的生物学问题是什么；
- 哪些数据可以使用；
- 哪个样本才是独立实验单位；
- 什么结果算成功；
- 哪些异常不能被自动删除；
- 最终结论需要什么证据支持。

这个比喻也有边界：Codex 不是一个固定能力水平的人类助手，它的能力会随模型、工具、上下文和权限变化。无论能力多强，最终分析仍需要来源、参数、日志和独立证据来验证。OpenAI 的官方说明同样把 Codex 定位为需要人类指导和复核的工具，而不是人类判断的替代品。

### 教程的目标也需要改变

在没有 AI 工具时，教程容易把大量篇幅用于要求学生记住函数名和命令参数。

在 Codex 时代，更重要的学习目标是：

- 知道一个分析流程为什么存在；
- 能描述输入、输出和成功标准；
- 能检查 AI 选择的软件是否匹配数据；
- 能识别实验设计、统计和生物学解释中的错误；
- 能要求 AI 保留代码、版本、日志和决策记录；
- 在 AI 结果与官方文档或原始论文冲突时，知道应该相信什么证据。

因此，这套教程不是要和 Codex 比谁写代码更快。它要帮助读者成为一个能够正确提出问题、监督分析、检查证据并承担科研判断的人。
