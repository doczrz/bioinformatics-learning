# Bioinformatics Interactive Learning

> 面向生物信息学初学者的开源双语交互式教材。

[中文](README.md) | [English](README.en.md)

Bioinformatics Interactive Learning 希望把生物信息学课程变成一个可以持续更新、自由访问并按自己的节奏学习的开源项目。它不是一组需要被动观看的幻灯片，而是一套可以在浏览器中阅读、切换语言、复制代码并连接外部数据来源的交互式教材。

## 当前状态

项目仍处于早期建设阶段。独立的双语教材阅读器已经可以本地运行，当前内置内容主要用于验证阅读与更新流程，完整课程正在持续编写中。

在集中建设阶段，我会尽量每 2–5 天发布一次教程更新；实际节奏可能根据内容审核、实验验证和开发安排调整。课程路线图描述的是计划覆盖的范围，不代表所有章节已经完成。

当前版本的边界：

- 支持中英文整页切换、章节导航、Markdown、代码块、图片和外部数据链接。
- 在线课程内容仓库尚未配置，“检查更新”会明确提示当前状态。
- 阅读教材不需要 ChatGPT 或 Codex 插件；网页本身不执行 R、Python 或 SSH 命令。

## 快速开始

下面以 Windows 10/11 为例，适合第一次使用命令行的读者。每条命令都应在“命令提示符”中单独输入，输入后按 `Enter` 执行。如果你已经安装了某个工具，可以跳过对应的安装步骤。

### 环境要求

- [Git](https://git-scm.com/install/windows)：从 GitHub 下载项目。
- [Node.js](https://nodejs.org/en/download)：运行网页开发工具；需要 20.19.0–20.x，或 22.12.0 及更高版本。新安装时建议选择当前的 LTS（长期支持）版本。
- [pnpm 10.24.0](https://pnpm.io/10.x/installation)：安装项目依赖并启动网页。本项目已锁定使用该版本。

### Windows：第一次安装

#### 1. 打开命令提示符

1. 同时按下键盘上的 `Win` 和 `R` 键。
2. 在“运行”窗口中输入 `cmd`。
3. 按 `Enter`，即可打开黑色的“命令提示符”窗口。

后面的命令都在这个窗口中运行，不要输入到浏览器地址栏，也不要复制命令前面的 `C:\Users\你的用户名>` 提示符。安装新工具后，请关闭旧的命令提示符窗口并重新打开，让 Windows 识别新安装的命令。

#### 2. 安装并检查 Git

1. 打开 [Git for Windows 官方下载页](https://git-scm.com/install/windows)。大多数 Windows 电脑选择 x64 安装程序；Windows on Arm 设备选择 ARM64。
2. 运行下载的安装程序。没有特殊需求时可以保留默认选项。
3. 重新打开命令提示符，然后运行：

```bat
git --version
```

如果看到类似 `git version 2.x.x` 的内容，说明 Git 已安装成功。

#### 3. 安装并检查 Node.js

1. 打开 [Node.js 官方下载页](https://nodejs.org/en/download)。
2. 选择当前的 LTS 版本和 Windows Installer（`.msi`）。本项目支持 Node.js 20.19.0–20.x，或 22.12.0 及更高版本。
3. 运行安装程序并保留默认选项。npm 会随 Node.js 一起安装；安装完成后重新打开命令提示符。
4. 依次运行：

```bat
node --version
npm --version
```

第一条命令应显示受支持的版本，例如 `v24.18.0`。如果使用 Node.js 20，版本不得低于 `v20.19.0`；如果使用 Node.js 22 或更高版本，版本不得低于 `v22.12.0`。第二条命令应显示 npm 的版本号。

#### 4. 安装并检查 pnpm

Node.js 安装成功后，在命令提示符中运行：

```bat
npm install --global pnpm@10.24.0
pnpm --version
```

第一条命令会从网络下载 pnpm，可能需要等待一段时间。第二条命令显示 `10.24.0` 即表示安装成功。如果仍提示“`pnpm` 不是内部或外部命令”，请关闭命令提示符，重新打开后再运行 `pnpm --version`。

### 本地运行

以下示例把项目下载到当前用户文件夹（通常类似 `C:\Users\你的用户名`）。请在命令提示符中每次运行一行：

```bat
cd /d "%USERPROFILE%"
git clone https://github.com/doczrz/bioinformatics-learning.git
cd bioinformatics-learning
pnpm --dir web install
pnpm --dir web dev
```

`git clone` 只需在第一次下载项目时运行，`pnpm --dir web install` 通常也只需在首次运行或依赖更新后运行。执行 `pnpm --dir web dev` 后，命令提示符会保持运行，并显示一个 Vite 本地地址，通常是 `http://localhost:5173/`。复制这个地址，在 Chrome、Edge 或其他浏览器中打开即可开始阅读。阅读期间不要关闭命令提示符；需要停止网页时，在该窗口中按 `Ctrl+C`。如果 Windows 询问是否终止批处理，请输入 `Y` 并按 `Enter`。

如果项目已经位于其他文件夹，请跳过 `git clone`，并把下面路径替换为项目的实际位置：

```bat
cd /d "C:\你的项目路径\bioinformatics-learning"
pnpm --dir web dev
```

如果 `git`、`node` 或 `pnpm` 再次出现“不是内部或外部命令”，先重新打开命令提示符，再运行相应的版本检查命令。若 Vite 使用的不是 `5173` 端口，请以命令提示符实际显示的地址为准。

### 以后再次打开教材

第一次安装完成后，Git、Node.js、pnpm 和已经下载的项目都会保留在电脑中，不需要重新安装，也不需要再次运行 `git clone`。以后每次阅读时：

1. 按 `Win+R`，输入 `cmd` 并按 `Enter`。
2. 在命令提示符中逐行运行：

```bat
cd /d "%USERPROFILE%\bioinformatics-learning"
pnpm --dir web dev
```

3. 等待命令提示符显示 `Local:` 地址，把该地址复制到浏览器中打开。
4. 阅读期间保持命令提示符窗口开启；结束后按 `Ctrl+C` 停止服务。

如果项目没有保存在默认位置，请把第一条命令中的路径换成项目的实际路径。只想继续阅读时，不需要再次运行 `pnpm --dir web install`。

如果想先获取 GitHub 上最新发布的程序和教材内容，再启动网页，请运行：

```bat
cd /d "%USERPROFILE%\bioinformatics-learning"
git pull
pnpm --dir web install
pnpm --dir web dev
```

`git pull` 显示 `Already up to date.` 表示当前已经是最新版本。更新后再次运行 `pnpm --dir web install`，可以确保新增或变更的项目依赖也被正确安装。

### 如何使用教材

- 使用页面顶部的“中文 / English”按钮切换整页语言。
- 使用左侧目录进入不同章节。
- 阅读 Markdown 内容、图片和代码示例，并按需复制代码。
- 通过教材提供的链接访问数据集原始来源。
- 使用“检查更新”查看是否有经过验证的新课程内容。

教材不会自动下载数据、运行分析命令或上传你选中的文字。遇到分析问题时，可以引用具体章节、代码和错误信息向 GPT 或其他 AI 助手提问。

## 为什么做这个项目

我第一次接触单细胞测序和转录组学是在 2020 年 3 月。当时，网络上很多讨论集中在如何从 TCGA 或 GEO 中挖掘公共数据、快速完成分组与分类，再形成一篇所谓的“纯生信”文章。

我也曾沿着这条路学习，接触过许多在线分析工具。因为急于提高学习速度，我踩过付费工具信息不透明、投入与收获不匹配的坑。这些经历让我一直在想：能不能做一套超越传统教材的交互式教程，让初学者少踩一些坑，更顺畅、更系统地掌握生物信息学的基础知识？

这个项目由此开始。

## 我期待的学习方式

未来的教材和大学课程不一定只能依赖大班课堂与一套精美但静态的 PPT。课程也可以成为一个开源项目，承载教师多年积累的理解、判断与实践经验，并让更多人公平地站在这些经验之上继续探索。

学生可以先按照自己的进度阅读、练习和记录问题，再由任课教师每周集中答疑。这样，课堂时间可以更多地用于讨论、纠错和理解真正困难的部分，而不是重复单向讲授。

我希望这个项目最终形成一套足以支撑大学课程的内容体系，也希望它能给其他教师和课程建设者一些启发：课件可以被持续维护，知识可以被共同修正，学习也可以更开放、更主动。

## 课程路线图

计划中的内容将从基础工具逐步延伸到多组学分析：

1. **计算基础**：R、Python、Linux 与常见生物信息学数据格式。
2. **转录组入门**：RNA-seq 与单细胞 RNA-seq 的发展历程，从 SRA/FASTQ 到表达矩阵，以及质量控制、标准化等基础流程。
3. **单细胞下游分析**：细胞聚类、分群与注释，细胞通讯、功能富集、反卷积及其他常见分析方法。
4. **Bulk 与代谢组学**：Bulk RNA-seq、GO/KEGG 富集分析及代谢组学相关流程。
5. **空间与多组学**：空间转录组、空间蛋白组、空间代谢组，以及不同组学之间的联合分析。

这份路线图会随着课程建设和社区反馈继续调整。每个正式章节都应尽量解释方法为什么成立、适用于什么场景、常见误区是什么，而不只是给出一段可以复制运行的代码。

## 参与贡献

项目负责人不可能精通每一种分析方法。如果你对三维基因组、空间组学、算法方法或其他生物信息学方向有深入理解，并愿意把自己的经验整理成可复用的教程，欢迎参与共同维护。

你可以：

- 通过 GitHub Issue 报告课程中的错误、过时内容或不清楚的表述。
- 提交 Pull Request 改进教材、代码示例、图表或数据来源。
- 先通过 Issue 提出新章节或课程合作建议，说明主题、目标读者和你希望贡献的内容。
- 分享适合共同开发的研究或教学项目。

对于教程本身的错误，欢迎直接批评和指正。对于个人分析任务的排错，请先查阅教材与工具官方文档，并优先使用 GPT 等 AI 助手整理问题；维护者暂时无法通过私信提供一对一分析支持。

## 构建与部署

生成生产版本：

```bash
pnpm --dir web typecheck
pnpm --dir web build
pnpm --dir web preview
```

生产文件位于 `web/dist/`，可以部署到支持 HTTPS 的静态网站托管服务。关于内容发布、缓存和网页执行边界，请参阅 [`web/README.md`](web/README.md)。
