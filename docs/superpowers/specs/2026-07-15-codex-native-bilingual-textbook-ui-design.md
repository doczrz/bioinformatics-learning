# Codex 原生双语交互教材 UI 设计

日期：2026-07-15  
状态：已获用户批准，等待书面规范复核  
项目根目录：`D:\bioinformatics`

## 1. 目标

构建一个在 Codex 中直接打开和使用的双语交互教材项目。教材以 Markdown、图片、代码和外部数据清单为内容载体，以 Codex 原生对话作为问答入口，并为未来的本地 R 与 SSH/HPC 执行能力保留稳定接口。

本设计只规定 UI、页面结构、状态边界和交互规则，不规定课程主题、章节数量、知识顺序、实例数据或具体代码内容。

## 2. 已确认的产品决策

1. 采用“方案一”：Codex 原生教材仓库加 Skill。
2. 不开发独立网页聊天框，不复制 Codex 的对话能力。
3. 用户在 Codex 原生对话中自由提问；可以引用章节、文件、代码或粘贴选中的文本。
4. 采用整页中文/英文切换，不使用逐段双语或左右双栏。
5. 教材正文保持单列阅读；Codex 对话、文件树和终端由 Codex 原生界面提供。
6. 本地 R 与 SSH/HPC 第一版只显示“未开放”或“接口保留”，不检测环境、不请求凭据、不运行命令。
7. 大型实例数据不进入教材仓库；用户从原始数据源自行下载。
8. 教材允许每 2–5 天更新一次，但任何更新都不得覆盖用户笔记、进度或已下载数据。

## 3. 交付边界

### 3.1 教材项目负责

- 课程首页与章节 Markdown。
- 中文和英文页面映射。
- 图片、图注、代码和权威来源。
- 外部数据清单与下载说明。
- 内容版本、变更摘要和更新状态页。
- Codex Skill 的课程导航、上下文读取和安全更新工作流。
- 用户进度与偏好的本地状态格式。

### 3.2 Codex 原生提供

- 用户对话与连续追问。
- 工作区、文件和仓库读取。
- 权限审批、终端和开发工具。
- 未来的本地 R 与 SSH/HPC 执行入口。

### 3.3 第一版不包含

- 自定义聊天框或浮动“问 Codex”按钮。
- 固定三栏宽度或对 Codex 应用外壳的修改。
- 教材内嵌大型数据文件。
- 自动开始数据下载。
- 本地 R、Python 或 Linux 命令执行。
- SSH 地址、用户名、密码、密钥或集群配置。
- 课程内容编排与具体生物信息学知识范围。

## 4. 信息架构

项目包含四个主要 UI 页面，均以 Codex 可读取的 Markdown 文件呈现。章节页与数据页属于发布内容；包含用户进度的课程首页和包含本地更新状态的状态页由受版本管理的模板生成，并被 Git 忽略，避免刷新本地状态时制造教材内容冲突。

```text
首次入口 OPEN_COURSE.md
├── 生成课程首页 START_HERE.md
│   ├── 章节阅读页 chapters/<language>/<lesson>.md
│   ├── 数据下载页 datasets/CATALOG.md
│   └── 生成更新与环境状态页 COURSE_STATUS.md
└── 初始化失败时的恢复说明
```

`OPEN_COURSE.md` 是无本地状态时的稳定入口。它指导 Codex 调用 Skill 初始化或刷新本地视图，然后打开 `START_HERE.md`。所有页面必须提供返回课程首页的明确链接。章节页还必须提供上一章、下一章和对应语言页面的链接。

## 5. 页面设计

### 5.1 课程首页：`START_HERE.md`

首页只承担四项任务：

1. 继续上次学习。
2. 浏览课程目录。
3. 切换整页语言。
4. 查看教材、本地 R 和 SSH/HPC 的状态。

推荐信息顺序：

1. 教材名称与一句话说明。
2. 中文/英文切换链接。
3. “继续学习”区，显示最后章节与本地进度。
4. 课程目录，显示完成、学习中和未开始状态。
5. 数据清单、代码索引、术语与来源入口。
6. 内容版本与最后更新时间。
7. 本地 R 和 SSH/HPC 的禁用状态。

原型中的按钮在实际 Markdown 中实现为清晰的文本链接或由 Skill 解释的 Codex 指令，不依赖自定义 JavaScript 控件。

### 5.2 章节阅读页：`chapters/<language>/<lesson>.md`

章节页采用稳定、单列的教科书结构：

1. 上一章、课程首页、下一章导航。
2. 中文/英文整页切换。
3. 章节阅读进度。
4. 章节编号、标题、摘要、预计阅读时间和更新日期。
5. 本章目标。
6. 分层正文。
7. 图片与可独立理解的图注。
8. 带语言和来源标识的代码块。
9. 代码后的参数解释。
10. 就近出现的注意、风险和版本差异。
11. 权威来源链接。
12. 底部上一章和下一章导航。

用户直接在 Codex 对话中提问。Skill 应指导 Codex 优先读取当前章节、相邻代码、图注和来源，再回答问题。Skill 不限制用户只能使用预设问题。

### 5.3 数据下载页：`datasets/CATALOG.md`

数据页只维护数据清单和下载指导。每个条目至少展示：

- 稳定数据 ID。
- 显示名称。
- 原始来源与登录号或永久链接。
- 用途说明。
- 文件类型。
- 预计下载大小。
- 预计解压后磁盘需求。
- 使用许可或访问限制。
- 校验算法与校验值（来源提供或项目维护时）。
- 原始来源链接。
- 可复制的下载命令（适用时）。

下载必须由用户主动发起。教材不得因为打开页面、进入章节或检查更新而自动下载数据。

### 5.4 更新与环境状态页：`COURSE_STATUS.md`

状态页展示：

- 当前内容版本。
- 可用的新版本。
- 最近检查时间。
- 变更摘要。
- 查看详细差异的入口。
- 请求用户确认的更新入口。
- 用户笔记、进度和下载数据的保护说明。
- 本地 R 与 SSH/HPC 的禁用状态。
- 离线、外链失效和本地修改等异常状态。

第一版不得展示可操作的 R 或 SSH 配置表单，以免用户误认为该能力已经可用。

## 6. 双语规则

1. 中文和英文使用独立 Markdown 文件。
2. 每对页面共享同一个稳定章节 ID。
3. 每页顶部提供对应语言页面的直接链接。
4. 两种语言共享图片、代码、数据 ID、来源和章节顺序。
5. 语言切换必须停留在同一章节，不跳回课程首页。
6. 缺少翻译时明确标记“该语言版本尚未发布”，不得静默跳到其他章节。
7. 用户语言偏好保存在本地状态中，不写入发布内容。

## 7. 仓库与状态隔离

建议的发布内容：

```text
course-project/
├── OPEN_COURSE.md
├── chapters/
│   ├── zh/
│   └── en/
├── figures/
├── code/
├── datasets/
│   ├── CATALOG.md
│   └── catalog.yml
├── ui/
│   └── templates/
│       ├── START_HERE.template.md
│       └── COURSE_STATUS.template.md
├── skill/
└── course-manifest.yml
```

用户本地状态：

```text
.course-state/
├── progress.json
├── preferences.json
├── update-cache.json
└── download-records.json

my-notes/
└── *.md
```

项目根目录中的 `START_HERE.md`、`START_HERE.zh.md`、`START_HERE.en.md`、`COURSE_STATUS.md`、`COURSE_STATUS.zh.md` 和 `COURSE_STATUS.en.md` 是由 Skill 根据模板与 `.course-state/` 生成的本地视图；它们与 `.course-state/`、`my-notes/` 一样必须被 Git 忽略。发布更新脚本不得删除 `.course-state/` 或 `my-notes/`，也不得把本地状态写回发布模板。

## 8. 状态模型

### 8.1 学习进度

进度使用稳定章节 ID，而不是标题或文件位置。最低状态集合为：

- `not_started`
- `in_progress`
- `completed`

首页和章节页中的进度展示由 Skill 脚本根据 `.course-state/progress.json` 生成或刷新到本地视图。若用户从未启用进度记录，页面应显示“未记录”，而不是虚构百分比。

### 8.2 用户偏好

`preferences.json` 至少允许保存：

- 默认语言。
- 最后访问章节 ID。

第一版不需要账号同步、跨设备同步、排行榜、连续学习天数或奖励系统。

### 8.3 更新状态

更新检查可自动执行，但更新应用必须由用户确认。更新前需要：

1. 获取远端版本信息。
2. 比较当前版本与远端版本。
3. 检查教材发布文件是否存在本地修改。
4. 展示内容变化摘要。
5. 请求用户确认。
6. 更新发布内容。
7. 验证用户目录未发生变化。
8. 从最新模板与本地状态重新生成 `START_HERE.md` 和 `COURSE_STATUS.md`。

## 9. 数据流

```text
作者更新教材仓库
        ↓
用户或 Skill 检查远端版本
        ↓
比较版本、文件差异和本地修改
        ↓
向用户展示更新摘要并请求确认
        ↓
更新教材发布内容
        ↓
保留 .course-state/、my-notes/ 和外部下载数据
        ↓
刷新课程首页与状态页
```

数据下载流与教材更新流完全独立：

```text
用户打开数据清单
        ↓
检查来源、许可、大小与空间需求
        ↓
用户主动打开原始来源或复制下载命令
        ↓
数据保存到用户选择的位置
        ↓
用户自行运行校验或请 Codex 协助检查
```

## 10. 错误与安全处理

### 10.1 离线

- 继续允许阅读当前教材。
- 标记无法检查更新或验证外链。
- 不把离线误报为“已是最新版”。

### 10.2 外部数据链接失效

- 保留数据 ID、来源、登录号和原有链接。
- 提示链接检查失败。
- 不自动切换到未经审核的镜像。

### 10.3 教材发布文件有本地修改

- 禁止自动覆盖。
- 展示差异并让用户选择保留、另存或在确认后更新。
- `my-notes/` 中的内容不计为冲突。

### 10.4 本地状态损坏

- 备份无法解析的状态文件。
- 重建空状态。
- 不修改教材、笔记或下载数据。

### 10.5 未开放的执行接口

- 本地 R 和 SSH/HPC 始终显示禁用状态。
- 不执行环境探测。
- 不请求或保存凭据。
- 不生成会被误认为已经提交的运行结果。

## 11. 视觉与可读性规则

- 正文保持单列，避免中英文左右挤压。
- 标题层级稳定，不使用过多颜色。
- 代码、图片、警告和来源紧邻其解释段落。
- 不使用无意义的操作按钮或预设提问按钮。
- 状态文字不能只依赖颜色，必须同时使用文字标签。
- 图片必须带图号、图题、替代文本和可独立理解的图注。
- 所有链接使用能描述目标的文字，不使用模糊的“点击这里”。
- 原型中的卡片和按钮仅表达信息层级；实际呈现服从 Codex Markdown 渲染能力。

## 12. 验证与验收

### 12.1 页面与导航

- `START_HERE.md` 能链接到全部发布章节。
- `OPEN_COURSE.md` 能在本地视图不存在时指导 Codex 初始化首页和状态页。
- 每章能返回首页并前后翻页。
- 每个已发布中文页面能找到对应英文页面，反之亦然。
- 缺失翻译时出现明确提示。
- 数据页和状态页均能返回首页。

### 12.2 内容组件

- 图片路径有效，并包含替代文本和图注。
- 代码块声明语言。
- 外部来源链接可被检查。
- 数据条目包含必需字段。
- 大型数据未进入 Git 历史。

### 12.3 状态与更新

- 空状态能够生成首页与状态页。
- 进度使用稳定章节 ID。
- 离线时教材仍可读。
- 有本地修改时更新被阻止。
- 更新前后 `my-notes/` 与 `.course-state/` 内容保持不变。
- 状态文件损坏时可安全重建。

### 12.4 执行接口

- 第一版任何页面或 Skill 工作流都不会运行 R、Python、Linux、SSH 或调度器命令。
- 第一版不会请求服务器凭据。
- 禁用状态在首页、章节页和状态页保持一致。

## 13. 后续阶段

在第一版 UI、内容系统和更新机制稳定后，后续阶段可以分别设计：

1. 本地 R 环境检测与批准后执行。
2. SSH/HPC 连接、远程目录与调度器工作流。
3. 可选的静态网页阅读展示层。
4. 插件化分发与更大范围的安装体验。

这些能力不属于当前设计的实施范围，也不得在第一版中以可点击但无效的配置功能出现。
